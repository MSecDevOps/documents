    API Reference for MSECURITY LMS

    This document outlines all API endpoints for the MSECURITY LMS platform, detailing their purpose, HTTP methods, required payloads, and sample responses.

    1. Authentication Endpoints

    1.1 Token Obtain

    Endpoint: /user/token/
    Method: POST
    Description: Generate access and refresh tokens for a user.
    Request:
    { "email": "user@example.com", "password": "yourpassword" }
    Response:
    { "access": "access_token", "refresh": "refresh_token" }

    1.2 Token Refresh

    Endpoint: /user/token/refresh/
    Method: POST
    Description: Refresh an access token using the refresh token.
    Request:
    { "refresh": "refresh_token" }
    Response:
    { "access": "new_access_token" }

    1.3 Register User

    Endpoint: /user/register/
    Method: POST
    Description: Register a new user account.
    Request:
    { "first_name": "John", "last_name": "Doe", "email": "user@example.com", "phone_number": "1234567890", "device_type": "mobile", "device_identifier": "device_uuid" }
    Response:
    { "message": "Registration successful. Check your email for login credentials." }

    1.4 Login User

    Endpoint: /user/login/
    Method: POST
    Description: Authenticate a user and retrieve tokens.
    Request:
    { "email": "user@example.com", "password": "yourpassword", "device_identifier": "device_uuid" }
    Response:
    { "message": "Login successful." }

    1.5 Logout User

    Endpoint: /user/logout/
    Method: POST
    Description: Log out the user and invalidate their token.
    Request:
    { "refresh_token": "refresh_token" }
    Response:
    { "message": "Successfully logged out." }

    1.6 Password Reset

    Endpoint: /user/password-reset-email/,
    Method: POST
    Description: Send a temporary password to the user's email.
    Request:
    { "email": "user@example.com" }
    Response:
    { "message": "Temporary password sent to your email." }

    2. Profile Management

    2.1 Get Profile

    Endpoint: /student/profile/<int:user_id>/
    Method: GET
    Description: Retrieve the logged-in user's profile.
    Response:
    { "email": "user@example.com", "full_name": "John Doe", "image": "profile_image_url", "country": "USA", "about": "Student at LMS" }

    2.2 Update Profile

    Endpoint: /student/profile/<int:user_id>/
    Method: PUT
    Description: Update profile information.
    Request:
    { "full_name": "John Doe Updated", "country": "Canada" }
    Response:
    { "message": "Profile updated successfully." }

    3. Course Management

    3.1 List All Courses

    Endpoint: /course/course-list/
    Method: GET
    Description: Retrieve all published courses.
    Response:

    [ { "id": 1, "title": "Cybersecurity Basics", "description": "Learn the basics of cybersecurity.", "price": 100.0, "image": "course_image_url" } ]

    3.2 Course Details

    Endpoint: /course/course-detail/<slug:slug>/
    Method: GET
    Description: Retrieve detailed information about a specific course.
    Response:

    { "title": "Cybersecurity Basics", "description": "Comprehensive course on cybersecurity.", "price": 100.0, "modules": [ { "title": "Introduction", "videos": [ { "title": "What is Cybersecurity?", "duration": "10:00" } ] } ] }

    3.3 Enroll in a Course

    Endpoint: /course/<slug:slug>/enroll/
    Method: POST
    Description: Enroll the logged-in user in a course.
    Response:

    { "message": "Enrolled successfully.", "course": "Cybersecurity Basics" }

    4. Payment Endpoints

    4.1 Create Payment

    Endpoint: /payme/create-payment/
    Method: POST
    Description: Initiate payment for a course.
    Request:

    { "order_oid": "order_uuid" }
    Response:
    { "payment_id": "payment_uuid", "payment_url": "http://payme.uz/payment_link" }

    4.2 Check Payment Status

    Endpoint: /payme/check-payment/<str:payment_id>/
    Method: GET
    Description: Retrieve the payment status.
    Response:
    { "status": "completed", "details": "Payment successful" }

    5. Gamification

    5.1 Mark Video as Completed

    Endpoint: /gamification/videos/complete/
    Method: POST
    Description: Mark a video as completed and reward points.
    Request:

    { "user_id": 1, "video_id": 101 }
    Response:
    { "message": "Video marked as completed.", "user": { "id": 1, "points": 100 } }

    6. Quiz and Test

    6.1 Retrieve Test

    Endpoint: /test/<int:video_id>/
    Method: GET
    Description: Retrieve test details for a specific video.
    Response:
    { "id": 1, "title": "Cybersecurity Quiz", "questions": [ { "id": 11, "text": "What is a firewall?", "answers": [ {"id": 101, "text": "Hardware", "is_correct": true}, {"id": 102, "text": "Virus", "is_correct": false} ] } ] }

    6.2 Submit Test

    Endpoint: /test/submit/
    Method: POST
    Description: Submit a completed test.
    Request:

    { "test": 1, "questions": [ {"question_id": 11, "selected_answer_id": 101} ] }
    Response:
    { "message": "Test successfully submitted", "score": 90 }
